  1. Overview of Git and GitHub

  a. Git:
  Git is a Distributed Version Control System (which can be called DVCS). Thanks to Git, the code management and developer teamwork have become simpler and more convenient. Also, Git provides each programmer with a separate repository containing a complete history of changes.
  These are the common benefits when using Git:

  • Easy to use, fast and very fast.
  • Easy to combine branches together, help code development in teamwork much simpler.
  • Can clone code source from the repository or clone a modified version from the repository, or a branch from a repository so that you can work anywhere.

  b. GitHub:
  GitHub is a project and code management system, with the purpose of acting like a social network for programmers. Also, GitHub is a popular service which can provide storage to save Git source code for software projects. Aside from having all Git features, GitHub also provides features about social so that the developers can easily communicate with each other.
  Social networking features that GitHub includes are feeds, feedbacks and network graph so that the developers can learn from each other through commit history.
  These are the reasons you should use GitHub:

  • Easy to manage source code.
  • Can track changes through versions
  • GitHub helps improving your coding skills, not to mention tracking bugs.
  • GitHub is a useful resource.

  In conclusion, Git and GitHub have become influential factors in the open source community. Nowadays, using Git and GitHub is mandatory if you want to apply for jobs. That is why you should learn about Git and GitHub now.

 

  2. How to install Git and GitHub

  There are two ways to install Git and GitHub:
  a. Install Git (and also GitHub) using GitHub Desktop:
  Step 1: Go to this website: https://desktop.github.com and download GitHub Desktop.
  Step 2: Open GitHub Desktop Setup and wait for GitHub to be installed.
  Note: installing GitHub Desktop will also install the latest version of Git if you have not downloaded Git yet.
  Step 3: Once GitHub Desktop opened, log in with your GitHub account and you can start using GitHub on your desktop.
  b. Install Git only: (this method is used only when you want to use GitHub directly on GitHub website and do not want to use GitHub on desktop):
  Step 1: Go to this website: https://git-scm.com/downloads
  Step 3: Open Git Setup and follow the instructions as provided in the Git Setup wizard screen until the installation is complete.
  Note: these instructions are most for Windows, as because Mac and Linux usually already have Git installed. However, if you did not have Git installed for whatever reason, you can install Git using the methods on top.

  To verify that Git has been installed, open up your terminal application and type ‘git version’ to see whether Git is installed or not and which version it is, or it will alert you that git is an unknown command.

 

  3. Some basic Git commands

  1/ git config: to set username and email in the main configuration file.
  To check username and email, type git config -- global user.name and git config -- global user.email.
  To set new username and email, type git config -- global user.name = “your_username” and git config -- global user.email = “your_useremail”.

  2/ git init: to create new git repository, project or repository which is already had on server.
  To do this, type git init in the project original folder.

  3/ git clone: to copy one git repository from the remote source.
  To do this, type git clone <(clone git url)>

  4/ git status: to check files status you have changed in your working directory.
  To do this, type git status

  5/ git add: to add the files and folders you want to the staging area.
  To do this, type git add * to add all files, type git add (file name) to add your selected file.

  6/ git commit: to save a snapshot about your changes in your working directory. And all the changed files, folders must be in the staging area. Every time commit it will save history changes with the username and email of who has commit. In Git, you can also restore these files in its commit history.
  To do this, type git commit --m ”(Message so that you can note the changes)”

  7/ git push: after you had added and committed the changes, you can do this to push all these changes to your local repository.
  To do this, type git push after git add and git commit.

  8/ git pull: to pull all the updates from your local repository and apply these updates on your code.
  To do this, type git pull

  9/ git branch: to list all the branches.
  To do this, type git branch

  10/ git checkout: to move to the other branch.
  To do this, type git checkout <(branch name)>

  11/ git stash: to save changes that you do not want to commit now.
  To do this, type git stash

  12/ git merge: to merge two branches together.
  To do this, move to the branch that you want to merge and type git merge <(branch that you want to merge with)>

  13/ git reset: to remove files and folders you have added to the staging area so that these files will not be committed.
  To do this, type git reset HEAD (file name)

 

  4. Branching

  Branch is a function use for branching and storing history changes. Branch which has been branched will not affect other branches so it can be modified in the same repository at the same time. Moreover, branches can be merged into one branch through using git merge.
  Branching will be used when a team member wants to create a branch for their own work and do not want to affect the work of other members. After that, members who have finished their works and changes will start adding these to the main branch. In this way, other member’s works will not be affected, and you can also do your own work. Furthermore, by saving works and changes history, if there is any conflicts or problems, it will be easier to solved through checking history.
  When committing for the first time in a repository, Git will create a branch automatically which is named main. So, every commit will be added to the main branch in the future until the user switches to another branch.
  Here are some commands about branching:

  • git branch: to check all the branches in a project. If there are no branches except from the main branch, then there will be no results.
    • Important note: before creating a new branch, you must commit the main branch so that Git will understand what the main branch is. If not, there will be an error. So first, commit then you can create a new branch.
  • To create a new branch, type git branch (new branch name)
  • To switch to the branch you just create, type git checkout (branch name)
  • If you want to delete a branch from Git, type git branch -d (branch name). But to do that, you must not in the branch you want to delete, so the best way is to switch to the main branch and then you can delete branch.
  • To switch back to the main branch, just type git checkout main
  • If you want to merge branches together, first switch to the branch you want to merge and type git merge (branch name).
